## Этап 1: Валидация российских номеров телефонов

### 1.1 Создание утилиты валидации

**Файл:** `src/utils/phoneValidation.ts`

**Функционал:**
- Проверка формата номера (должен начинаться с +7 или 7)
- Проверка длины (11 цифр для российских номеров)
- Проверка кода оператора (валидные коды: 900-999, 800-899, 700-799, и т.д.)
- Нормализация номера (приведение к формату +7XXXXXXXXXX)
- Валидация перед отправкой SMS

**Пример реализации:**
```typescript
export const normalizePhone = (phone: string): string | null => {
  // Удаляем все символы кроме цифр и +
  const cleaned = phone.replace(/[^\d+]/g, '');
  
  // Если начинается с 8, заменяем на +7
  if (cleaned.startsWith('8')) {
    return '+7' + cleaned.slice(1);
  }
  
  // Если начинается с 7, добавляем +
  if (cleaned.startsWith('7') && !cleaned.startsWith('+7')) {
    return '+7' + cleaned.slice(1);
  }
  
  // Если уже начинается с +7
  if (cleaned.startsWith('+7')) {
    return cleaned;
  }
  
  return null;
};

export const validateRussianPhone = (phone: string): { valid: boolean; error?: string } => {
  const normalized = normalizePhone(phone);
  
  if (!normalized) {
    return { valid: false, error: 'Некорректный формат номера' };
  }
  
  // Проверка длины (должно быть +7 + 10 цифр = 12 символов)
  if (normalized.length !== 12) {
    return { valid: false, error: 'Номер должен содержать 10 цифр после +7' };
  }
  
  // Проверка кода оператора (вторая и третья цифры после +7)
  const operatorCode = normalized.substring(2, 4);
  const validCodes = ['90', '91', '92', '93', '94', '95', '96', '97', '98', '99', 
                      '80', '81', '82', '83', '84', '85', '86', '87', '88', '89',
                      '70', '71', '72', '73', '74', '75', '76', '77', '78', '79'];
  
  if (!validCodes.includes(operatorCode)) {
    return { valid: false, error: 'Некорректный код оператора' };
  }
  
  return { valid: true };
};
```

## Этап 2: Настройка Supabase для SMS

### 2.1 Настройка Phone Auth в Supabase Dashboard

**Шаги:**
1. Зайти в Supabase Dashboard → Authentication → Settings
2. Включить Phone Auth:
   - Enable phone signup: ON
   - Enable phone login: ON
3. Настроить SMS Provider:
   - Provider: Custom (для использования webhook)
   - Enable Send SMS hook: ON
   - Hook type: HTTPS
   - URL: `https://your-vercel-app.vercel.app/api/send-sms`
   - Secret: Сгенерировать секретный ключ

### 2.2 Настройка Rate Limiting

**В Supabase Dashboard:**
- Authentication → Settings → Rate Limits
- Настроить лимиты для:
  - SMS отправки: максимум 3 SMS в час на один номер
  - OTP проверки: максимум 5 попыток в 15 минут

### 2.3 Создание Edge Function для дополнительной валидации (опционально)
